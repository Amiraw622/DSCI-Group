1. Data Description¶

1.1 Dataset anaylze¶

Dataset: Players
number of observations: 196
number of variables: 7
Variable              type           meaning
experience     |     Factor      |  The experience of the player
subscribe      |     Boolean     |  If the player subscribe or not
hashedEmail    |     Factor      |  The email of the player
played_hours   |     Double      |  How many hours player played
name           |     Factor      |  The name of the player
gender         |     Factor      |  The gender of the player
Age            |     Numeric     |  The age of the player

1.1.2 Dataset issue¶

1) Some variables like name may not be useful
2) Some data may contain the missing data
3) There may be something wrong with the data like played_hours = 0

1.2.1 Dataset anaylze¶

Dataset: sessions
number of observations: 1535
number of variables: 5
Variable                     type           meaning
hashedEmail           |     Factor      |  The email of the player
start_time            |     POSIXct     |  The time when player start to play

original_start_time   |     Numeric     |  The original start time of the player
original_end_time     |     Numeric     |  The original end time of the player

1.2.1 Dataset issue¶

1) Original_start_time and original_end_time are numeric and in some cases equal
2) Some of the start time is equal to the end time
3) Some players may have many sessions

2. Questions¶

Broad Question:¶

What player characteristics and behaviours are most predictive of subscribing to a game-related newsletter, and how do these features differ between various player types?

3.5 Insights¶

• Non-subscribers are significantly concentrated in the shorter game duration range.
• Subscribers account for a higher proportion of longer game duration.
• The number of non-subscribers is significantly higher in the younger age groups.
• The proportion of subscribers is relatively higher in the older age groups

4. Methods and Plan¶

I propose using the K-Nearest Neighbors (KNN) classification algorithm.

In [ ]:

4.1 Why is this method appropriate?¶

Knn is simple and easy to understand, making it intuitive to use. It’s chosen because it doesn't require complex mathematical models and can produce relationships between player characteristics and subscription well.

4.2 Which assumptions are required, if any, to apply the method selected?¶

In Knn, we need to use distance to measure, so we need to put variable on same scale to make it easier for us to anaylze.If some features have much larger ranges, we will need to standardize them.

4.3 What are the potential limitations or weaknesses of the method selected?¶

The performance of KNN depends on choosing the right k value. If k is not suitable, the model may be hard to visualize.
With many features, the distances between data points become less meaningful, making KNN less effective for high-dimensional data.
Also, because we need to calculte the distance so it is slow.

4.4 How are you going to compare and select the model?¶

We will use cross-validation to test the model and tune the k value. We’ll also evaluate the model using precision to ensure it correctly predicts subscriptions.

4.5 How are you going to process the data to apply the model?¶

We’ll split the data into a training set and a test set to evaluate the model’s performance. We may also use cross-validation to improve the model’s accuracy.
